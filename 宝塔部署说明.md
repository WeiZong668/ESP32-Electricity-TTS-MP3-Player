# ESP32 文字转语音服务 - 宝塔部署教程

## 📦 服务说明

这是一个简单的文字转语音 HTTP 服务，专门为 ESP32 + MAX98357A 播放器设计。
- 输入文字，自动生成 MP3
- 提供 HTTP 链接供 ESP32 播放
- 自动缓存，相同文字不重复生成

---

## 🚀 宝塔面板部署步骤

### 1️⃣ 安装 Python 环境

在宝塔面板：
1. 进入 **软件商店**
2. 搜索并安装 **Python 项目管理器**
3. 安装 **Python 3.8** 或更高版本

### 2️⃣ 上传文件

1. 在宝塔 **文件管理** 中创建目录：
   ```
   /www/wwwroot/tts_server
   ```

2. 上传 `tts_server.py` 到该目录

### 3️⃣ 安装依赖

在宝塔 **终端** 或 SSH 中执行：

```bash
cd /www/wwwroot/tts_server

# 安装依赖包
pip3 install flask gtts
```

### 4️⃣ 创建 Python 项目

1. 打开 **Python 项目管理器**
2. 点击 **添加项目**
3. 填写信息：
   - **项目名称**：`tts_server`
   - **项目路径**：`/www/wwwroot/tts_server`
   - **启动文件**：`tts_server.py`
   - **端口**：`5000`（或其他未占用端口）
   - **Python 版本**：选择已安装的 Python 3.x

4. 点击 **提交**

### 5️⃣ 启动服务

在 Python 项目管理器中：
1. 找到刚创建的 `tts_server` 项目
2. 点击 **启动** 按钮
3. 查看状态是否为 **运行中**

### 6️⃣ 配置防火墙

在宝塔 **安全** 中：
1. 添加端口规则：`5000`
2. 放行该端口

在服务器防火墙（如阿里云、腾讯云安全组）：
1. 添加入站规则
2. 开放 TCP 端口 `5000`

http://127.0.0.1:5002

你会看到一个漂亮的网页界面。

## 📝 使用方法

### 方法 1：网页界面使用

1. 打开 `http://127.0.0.1:5002`
2. 在文本框输入要转换的文字
3. 点击 **生成语音**
4. 复制生成的 HTTP 链接
5. 粘贴到 ESP32 代码的 `MP3_URL` 中

### 方法 2：直接 API 调用

```bash
curl -X POST http://127.0.0.1:5002/api/tts \
  -H "Content-Type: application/json" \
  -d '{"text":"你好，欢迎使用ESP32播放器"}'
```

返回：
```json
{
  "success": true,
  "url": "/audio/xxxxx.mp3",
  "text": "你好，欢迎使用ESP32播放器"
}
```

完整播放链接：
```
http://127.0.0.1:5002/audio/xxxxx.mp3
```

## 🛠️ 进阶配置

### 修改端口

编辑 `tts_server.py` 最后一行：
```python
app.run(host='0.0.0.0', port=5000, debug=False)
```
改为你想要的端口，如 `8080`

### 修改缓存目录

编辑 `tts_server.py` 第 18 行：
```python
CACHE_DIR = "/tmp/tts_cache"
```
改为你想要的目录，如 `/www/wwwroot/tts_cache`

### 设置开机自启

在宝塔 Python 项目管理器中：
1. 找到项目
2. 点击 **设置**
3. 勾选 **开机自启**

## 🐛 常见问题

### 1. 服务无法启动

检查依赖是否安装：
```bash
pip3 list | grep -E "flask|gtts"
```

### 2. 无法访问网页

- 检查防火墙是否开放端口
- 检查服务是否运行：`ps aux | grep tts_server`
- 查看日志：在宝塔 Python 项目管理器中查看日志

### 3. 生成语音失败

- 检查服务器网络是否正常（gTTS 需要访问 Google）
- 如果无法访问 Google，可以考虑使用国内 TTS 服务（需修改代码）

### 4. ESP32 无法播放

- 确保使用 HTTP 而非 HTTPS
- 检查 URL 是否正确
- 确保服务器和 ESP32 在同一网络或服务器有公网 IP
